Cloud Computing

  • On-demand usage & delivery of IT resources
  • Examples:
    • Rent a server & serve a website
    • Rent file storage volumes
  • Flexible usage of compute power & data storage capabilities
  • No need to provision and maintain your own data center
  • Only pay for the resources you use, when you use them

On-premises advantages & disadvantages

Advantages:

  1. You have full control over your physical infrastructure & hardware.
  2. You know exactly where your computers (and data) are.

Disadvantages:

  1. You are responsible for maintaining the infrastructure.
  2. You are responsible for (long-term) capacity planning & upgrading.
  3. You are responsible for securing the infrastructure.
  4. You can’t react quickly to workload spikes (e.g., more requests).
  5. You also pay for idle resources.
  6. You are typically stuck to one or a few locations.

Cloud Advantages

  1. Reliability
    1. Rely on AWS
      • AWS is responsible for managing the underlying infrastructure.
      • Service Level Agreements (SLA) are available for many key services.
    2. Build Reliable Applications
      • Various services help you build reliable applications.
      • Your workload performs its intended function correctly & consistently.
    3. Rely on Global Reach
      • Fall back to other regions or go with a multi-region setup right from the start.
      • Move your workload within minutes to hours, instead of days or weeks.
  2. Agility, Elasticity & Scalability
    1. Agility
      • Use cloud resources within seconds or minutes.
      • Launch resources with a click or command.
    2. Elasticity
      • Start using more or less resources, just as needed.
      • No long-term planning required.
    3. Scalability
      • Scale up or down, as required.
      • Use auto-scaling services to reduce manual workload.
  3. Pay-as-you Go
    • You only pay for what you use.
    • There are a few upfront or subscription payment models.
  4. Global Reach
    1. Choose perfect location
    2. Spread our workloads to ensure high availability

Regions & Availability Zones

  • Region: A region in AWS terminology is a specific geographical area where AWS operates. Each region is entirely independent of the other and is designed to be completely isolated from failures in other regions. This isolation helps achieve the greatest possible fault tolerance and stability.
  • Availability Zones (AZs): Within each region, AWS operates several isolated locations known as Availability Zones. Each AZ is essentially a data center or a collection of data centers that are equipped to provide services. A key point about these AZs is that they are physically separated from each other within the region but are interconnected through low-latency links. This setup enhances both fault tolerance and performance.
    • Minimum AZs: Each region has at least three Availability Zones, enhancing the reliability of the services offered by providing backup in case one or two zones encounter issues.
    • Individual Data Centers: Each Availability Zone contains at least one data center, if not more.
  • Local Zones: These are extensions of a main AWS region that place compute, storage, database, and other select services closer to end-users, providing them with lower-latency access.
  • Edge Locations: These are sites deployed in major cities and highly populated areas worldwide to deliver content to end users with lower latency. They are part of AWS's content delivery network, Amazon CloudFront.
  • Wavelength Zones: These are AWS infrastructure deployments that embed AWS compute and storage services within the telecommunications providers’ datacenters at the edge of the 5G network, so application traffic from 5G devices can reach server applications without leaving the telecommunications network.
  • AWS Outposts: These bring native AWS services, infrastructure, and operating models to virtually any data center, co-location space, or on-premises facility for a truly consistent hybrid experience.

Self-Service vs Managed Services

AWS offers two main kinds of services

Self-Service:

  • Do it yourself: Users handle the setup and management of services.
  • Fully configurable: Users have complete control over the configuration.
  • Underlying hardware is still managed by AWS: AWS takes care of the physical infrastructure.

Managed Services:

  • AWS manages the hard parts: AWS handles the more complex aspects of service management.
  • Partially configurable: Users have some options for configuration, but not as extensive as with self-service.
  • Can be used together with other services (incl. self-service): Managed services can be integrated with both other managed services and self-service options.
